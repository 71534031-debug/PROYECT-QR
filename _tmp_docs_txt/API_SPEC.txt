API_SPEC.md
Descripción General
Esta especificación define los endpoints del backend del sistema de certificados digitales.
La API sigue un estilo REST y permite gestionar autenticación, actividades, participantes, certificados y validación pública.
MÓDULO: AUTENTICACIÓN
🔹 POST /api/auth/login
Descripción: Permite autenticar usuarios internos. 
Autenticación: ❌ No requerida 
Body
{
  "email": "string",
  "password": "string"
}
Response (200)
{
  "success": true,
  "token": "jwt_token",
  "user": {
    "id": 1,
    "nombre": "string",
    "rol": "ADMIN | ADMINISTRATIVO"
  }
}
Errores
401 → Credenciales inválidas 
403 → Usuario inactivo 
500 → Error del servidor 
MÓDULO: ACTIVIDADES
🔹 POST /api/actividades
Descripción: Registrar actividad académica 
Autenticación: ✅ Requerida 
Body
{
  "nombre": "string",
  "descripcion": "string",
  "fecha_inicio": "date",
  "fecha_fin": "date"
}
Response (201)
{
  "success": true,
  "data": {
    "id": 1
  }
}
Errores
400 → Datos inválidos 
409 → Actividad duplicada 
401 → No autorizado 
🔹 GET /api/actividades
Descripción: Listar actividades 
Autenticación: ✅ 
Response (200)
{
  "success": true,
  "data": []
}
MÓDULO: PARTICIPANTES
🔹 POST /api/participantes
Descripción: Registrar participante manual 
Autenticación: ✅ 
Body
{
  "nombres": "string",
  "apellidos": "string",
  "email": "string",
  "documento_identidad": "string",
  "actividad_id": 1
}
Response (201)
{
  "success": true,

    "id": 1

Errores
400 → Datos inválidos

 MÓDULO: PLANTILLAS
🔹 POST /api/plantillas
Descripción: Crear plantilla (sin contenido HTML — se usa imagen + campos)
Autenticación: ✅ 
 Body
{
  "nombre": "string"
}
 Response (201)
{
  "success": true,
  "data": {
    "id": 1
  }
}
🔹 GET /api/plantillas
Descripción: Listar plantillas 
Autenticación: ✅ 
🔹 GET /api/plantillas/:id
Descripción: Obtener plantilla con sus campos posicionables 
Autenticación: ✅ 
 Response (200)
{
  "success": true,
  "data": {
    "id": 1,
    "nombre": "string",
    "imagen_fondo": "ruta/imagen.png",
    "campos": []
  }
}
🔹 PUT /api/plantillas/:id/campos
Descripción: Guardar/actualizar posiciones de campos de una plantilla 
Autenticación: ✅ 
 Body
{
  "campos": [
    {
      "placeholder": "NOMBRE_COMPLETO",
      "x": 50, "y": 30,
      "font_size": 24, "alignment": "center",
      "color": "#1a1a2e", "width": 400, "height": 40,
      "orden": 1
    }
  ]
}
 Response (200)
{
  "success": true,
  "message": "Campos guardados correctamente"
}
Errores
400 → Faltan placeholders requeridos (lista de 10)
404 → Plantilla no encontrada
🔹 GET /api/plantillas/:id/campos
Descripción: Listar los campos configurados de una plantilla 
Autenticación: ✅ 
 Response (200)
{
  "success": true,
  "data": [ ... ]
}
🔹 POST /api/plantillas/:id/imagen
Descripción: Subir imagen de fondo para la plantilla (PNG/JPG, máx 10 MB)
Autenticación: ✅ 
 Body (multipart/form-data)
{
  "imagen": "file"
}
 Response (200)
{
  "success": true,
  "imagen_fondo": "uploads/images/plantilla_uuid.ext"
}
Errores
400 → Archivo no enviado o formato inválido
404 → Plantilla no encontrada
 MÓDULO: CERTIFICADOS
🔹 POST /api/certificados/generar
Descripción: Generar certificados automáticamente 
Autenticación: ✅
