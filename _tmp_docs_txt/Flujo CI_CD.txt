CICD
 Flujo CI/CD del Proyecto
Flujo completo
Developer → Git Push → Repositorio → CI Checks → Build → Deploy → Producción
Proceso paso a paso
El desarrollador realiza cambios en el código 
Hace commit y push al repositorio (branch main o develop) 
Se ejecutan validaciones automáticas (CI) 
Si todo es correcto: 
Se genera el build 
Se despliega automáticamente 
El sistema queda disponible en producción 
 Checks de Integración Continua (CI)
Antes del despliegue, el código debe pasar:
🔹 1. Linting
Validación de calidad de código 
Herramienta sugerida: 
ESLint 
npm run lint
🔹 2. Tests
Actualmente el proyecto no tiene tests automatizados definidos, pero se considera:
Validación manual (Postman, navegador) 
Flujo crítico probado: 
Login 
Generación de certificados 
Validación QR 
Futuro:
npm test
🔹 3. Build
Frontend:
npm run build
Backend:
Validación de ejecución sin errores

 Estrategia de Deploy
🔹 Branching

🔹 Flujo de despliegue
feature/* → develop → staging 
develop → pruebas → merge a main 
main → producción automática 
Consideraciones Importantes
Uso obligatorio de HTTPS en producción 
Variables de entorno configuradas en Vercel y Render 
No subir .env al repositorio 
Validar build antes de deploy 
Logs activos en backend (Render) 
 Errores Comunes en CI/CD
Error: Build falla en Vercel
Causa: dependencias o errores en código
Solución:
npm install
npm run build
Error: Backend no inicia en Render
Causa: puerto incorrecto
Solución:
Usar variable PORT 
No hardcodear puertos 
Error: Variables de entorno no funcionan
Causa: no configuradas en plataforma
Solución:
Agregar manualmente en: 
Vercel dashboard 
Render dashboard 
Error: CORS en producción
Causa: dominios no permitidos
Solución:
Configurar middleware CORS en backend
