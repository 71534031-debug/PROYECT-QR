MASTER_SPEC
Resumen del Sistema
Sistema web para la generación y validación segura de certificados digitales, que automatiza el registro de actividades, carga de participantes y emisión de certificados en PDF con código único y QR.
Permite validación pública en línea, garantizando autenticidad, trazabilidad y eficiencia operativa.
Está diseñado bajo arquitectura cliente-servidor y orientado a procesos institucionales del Colegio de Ingenieros del Perú – CD Huancavelica.

Restricciones Técnicas Importantes
Arquitectura cliente-servidor en capas (N-tier) 
Backend monolítico modular 
Uso obligatorio de React, Node.js y MySQL 
Dependencia de servicios en la nube (Vercel y Render) 
Sistema 100% web (no offline) 
Sin microservicios en esta versión 
Generación de certificados debe soportar emisión masiva (~100 registros) 
Tiempo de respuesta máximo: 3 segundos 
Seguridad básica: autenticación + HTTPS + control de acceso 
 Convenciones de Código
API basada en REST 
Uso de JSON para intercambio de datos 
Estructura modular por dominios (auth, actividades, certificados, etc.) 
Nombres de endpoints en snake_case o kebab-case consistentes 
Identificadores únicos para certificados (no reutilizables) 
Validaciones obligatorias en backend (no confiar en frontend) 
Manejo de errores estandarizado (HTTP status codes) 
Separación de capas: 
Controladores (request/response) 
Servicios (lógica de negocio) 
Modelos (base de datos) 
Logs para trazabilidad (acciones críticas)
